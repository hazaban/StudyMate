## 产品概述

**StudyMate 学习星球** 是一款基于 AI 和艾宾浩斯遗忘曲线的备考工具。它帮助你：

- **制定计划**：创建多科备考计划，按科目 + 章节拆分到每天
- **执行任务**：每日任务看板 + 番茄钟计时，自动记录实际用时
- **科学复习**：知识卡片和错题本按记忆曲线自动安排复习时间
- **AI 助手**：智谱 GLM AI 生成备考计划、每日任务、复习卡片，还能识别教材目录图片
- **游戏化激励**：完成学习种植作物，收获升级
- **统计分析**：学习时长饼图、趋势图、科目分布

### AI Provider

后端使用**智谱 GLM**（OpenAI 兼容协议），通过环境变量配置：

| Provider | 模型 | 用途 |
|----------|------|------|
| **智谱 GLM-4.5-Air** | `glm-4.5-air` | 纯文本对话、工具调用、Agent、计划生成、任务拆解、卡片生成、复盘总结 |
| **智谱 GLM-4.1V-Thinking-FlashX** | `glm-4.1v-thinking-flashx` | 图片识别、视频理解、GUI 理解（教材目录识别等） |
| **mock** | 内置假数据 | 未配置 API Key 时自动降级 |

> 申请地址：https://open.bigmodel.cn/

## 3. 番茄钟

### 布局概览

1. **关联任务**：点击可选择今日任务，计时时显示任务名
2. **计时圆环**：渐变圆环显示进度（红色=专注，绿色=休息）

### 完成通知

番茄钟完成一轮后：
- **Web Audio API**：播放上行音阶（C5→E5→G5）

- **浏览器通知**：页面在后台时弹出系统通知

## 6. 复习模式（卡片和错题共用）

### 卡片复习

1. 点击「开始复习」进入 → 显示进度：**3 / 10**
2. 每次显示一张卡片：科目标签 + Q（文字+图片）
3. 点击「点击查看答案」→ 显示 A（文字+图片）
4. 底部反馈：😣 未掌握(明天复习) / 🤔 较熟悉(3天后) / 😎 已掌握(7天后)
5. 复习完 🏆 "复习完成！"

### 错题复习

1. 类似卡片，显示 Q（题目+图片）
2. 查看答案后额外显示错误分析
3. 反馈：❌ 做错了(清零重来) / ✅ 做对了(连续3次标记已掌握)
4. 复习完 🏆 "复习完成！" + 正确/总题数

## 11. AI 智能规划

### AI Provider

后端通过智谱 GLM 提供 AI 能力（OpenAI 兼容协议）：

```
GLM_API_KEY 已配置 → 用智谱 GLM
否则 → 用 mock（内置假数据，演示用）
```

可强制指定：`AI_PROVIDER=glm|mock`

### 模型选型

| 场景 | 模型 | 特点 |
|------|------|------|
| 纯文本对话、工具调用、Agent | `glm-4.5-air` | 激活参少、便宜、速度快 |
| 传图 / 传视频 / GUI 理解 | `glm-4.1v-thinking-flashx` | 便宜、极速、多模态 |

### 两个模式

**1. 整体规划对话**
- 输入考试信息，AI 生成包含阶段划分、每周目标、每日时间分配、复习策略的完整计划
- 满意后点击「确认并应用计划」直接创建

**2. 科目框架分析**
- 选择科目 → 上传教材目录图片 → GLM-4.1V 视觉模型识别章节
- AI 给出每章建议学习时间和预计天数
- 确认后一键生成任务

### AI 接口列表

| 接口 | 功能 | 模型 |
|------|------|------|
| `POST /api/plans/ai/generate` | 生成完整学习计划 | GLM-4.5-Air |
| `POST /api/tasks/ai/generate` | 生成今日任务 | GLM-4.5-Air |
| `POST /api/cards/ai/generate` | 从学习内容生成卡片 | GLM-4.5-Air |
| `POST /api/ai/review` | 生成每日复盘总结 | GLM-4.5-Air |
| `POST /api/ai/syllabus` | 识别教材目录图片 | GLM-4.1V-Thinking-FlashX |

> 未配置 GLM_API_KEY 时，文本生成类接口使用内置 Mock 数据演示功能；图片识别接口返回 mock 的章节列表。

## 14. 艾宾浩斯记忆算法

### 知识卡片

| 掌握程度 | 第1次 | 第2次 | 第3次 | 第4次 | 第5次 | 第6次+ | 长期 |
|---------|------|------|------|------|------|--------|------|
| 未掌握 | 1天 | 1天 | 2天 | 3天 | 5天 | 8天 | 30天 |
| 较熟悉 | 3天 | 5天 | 8天 | 14天 | 21天 | 30天 | - |
| 已掌握 | 7天 | 14天 | 30天 | 30天 | 30天 | 30天 | - |

**规则**：
- 复习时回答正确 → 掌握程度升一级（未掌握→较熟悉→已掌握）
- 回答错误 → 降一级（已掌握→较熟悉→未掌握）
- 下次复习日期 = 上次复习日期 + 对应天数
- 系统保证**下次复习日期永不低于创建日期**

### 错题本

| 连续正确次数 | 下次复习间隔 |
|------------|------------|
| 0次（刚做错） | 明天 |
| 第1次正确 | 1天后 |
| 第2次正确 | 3天后 |
| 第3次正确 | 7天后 |
| 第4次正确 | 14天后 |
| 第5次正确 | 30天后 |

**规则**：
- 做错时正确次数清零，错误次数+1
- 连续正确 ≥3 次 → 自动标记为"已掌握"

## 15. 多端同步与部署架构

### 当前部署架构

```
用户手机/电脑浏览器
        ↓
Cloudflare Pages 前端（静态 H5 + _worker.js API代理）
        ↓ /api
Vercel 后端（Python Serverless, FastAPI + Mangum）
        ↓
Supabase PostgreSQL + 腾讯云 COS
```

- **前端**：Cloudflare Pages，仓库路径 `frontend/`，自动从 GitHub 构建
- **后端**：Vercel Serverless Functions，仓库路径 `backend/`，自动从 GitHub 构建
- **数据库**：Supabase 免费 PostgreSQL（500MB）
- **图片**：腾讯云 COS（可选）
- **API 代理**：Cloudflare Worker（_worker.js）绕过 Vercel 国内 DNS 污染

### 项目结构

```
StudyMate/
├── README.md
├── FEATURES.md
├── PROJECT_PLAN.md
├── docker-compose.yml
├── docs/                  # 部署文档

    ├── main.py            # 应用入口
```
