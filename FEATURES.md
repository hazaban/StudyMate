## 产品概述

**StudyMate 学习星球** 是一款基于 AI 和艾宾浩斯遗忘曲线的备考工具。它帮助你:

- **制定计划**:创建多科备考计划,按科目 + 章节拆分到每天
- **执行任务**:每日任务看板 + 番茄钟计时,自动记录实际用时
- **科学复习**:知识卡片和错题本按记忆曲线自动安排复习时间
- **AI 助手**:AI 生成备考计划、每日任务、复习卡片,还能识别教材目录图片
- **游戏化激励**:完成学习种植作物,收获升级
- **统计分析**:学习时长饼图、趋势图、科目分布

### AI Provider

后端支持三种 AI 服务,通过环境变量自动选择(优先级:混元 > DeepSeek > mock):

| Provider | 用途 | 接入方式 |
|----------|------|----------|
| **腾讯混元** | 文本生成(计划/任务/卡片/复盘) | OpenAI 兼容协议,`https://api.hunyuan.cloud.tencent.com/v1` |
| **DeepSeek** | 文本生成(备用) | OpenAI 兼容协议,`https://api.deepseek.com/v1` |
| **通义千问 Vision** | 图片识别(教材目录) | OpenAI 兼容协议,`https://dashscope.aliyuncs.com/compatible-mode/v1` |
| **mock** | 文本生成(无 Key 时) | 内置假数据,演示用 |

## 2. 任务看板

### 布局概览

1. **顶部统计**:已完成 / 待完成 / 总任务
2. **视图切换**:今日(默认)/ 历史(日历视图)
3. **日历**(历史模式):月视图日历,有任务的日期显示圆点标记
4. **筛选标签**:全部 / 待完成 / 已完成
5. **科目筛选**:横向滚动,可选择全部或特定科目
6. **任务列表**:每条显示复选框、内容、科目、章节、估计/实际用时、🍅 按钮
7. **番茄记录区**:选中日期若有专注记录,显示总次数和分钟数

### 功能清单

| 功能 | 操作 | 说明 |
|------|------|------|
| 添加任务 | 点击「+ 添加任务」 | 弹出添加弹窗 |
| 编辑任务 | 点击任务内容区域 | 弹出编辑弹窗(同首页弹窗) |
| 删除任务 | 点击任务卡片 | 弹出编辑弹窗后可删除 |
| 完成任务 | 点击 ○ | 完成后自动给农场对应科目施肥 |
| 取消完成 | 再次点击 ✓ | 恢复为待完成 |
| 跳转番茄钟 | 点击 🍅 | 自动关联该任务到计时器 |
| 查看历史 | 切换到「历史」标签 | 显示日历视图 |
| 选择日期 | 点击日历格子 | 加载该日期的任务和番茄记录 |
| 切换月份 | 点击 ‹ › 箭头 | 浏览不同月份 |
| 筛选任务类型 | 点击标签(全部/待完成/已完成) | 过滤列表 |
| 筛选科目 | 点击科目标签 | 只看该科目的任务 |
| 管理科目 | 点击 ⚙ 齿轮 | 弹出科目管理弹窗 |

### 循环任务

创建任务时可选择重复类型:
- **不循环**:仅在指定日期出现
- **每天**:从创建日起每天都会出现
- **工作日**:周一至周五出现
- **节假日**:周六日出现

循环任务完成后在 `completed_dates` 列表中记录完成日期,下次该日期不会再重复完成。

### 科目管理弹窗

- 列出所有科目,标注"预设"或"自定义"
- 预设科目(7个考研科目):不可删除
- 自定义科目:可删除(有二次确认)
- 底部输入框 + 添加按钮:新增自定义科目
- 新增的科目会保存到后端数据库,跨设备同步

## 3. 番茄钟

### 布局概览

1. **关联任务**:点击可选择今日任务,计时时显示任务名
2. **计时圆环**:渐变圆环显示进度(红色=专注,绿色=休息)
3. **时间显示**:MM:SS 大数字
4. **状态标签**:准备专注 / 专注中 🔥 / 休息中 ☕ / 已暂停
5. **控制按钮**:开始专注 / 暂停 / 重置
6. **时间设置**:专注时长(5-120分钟,默认25分钟)、休息时长(1-30分钟,默认5分钟)
7. **统计**:今日完成番茄数、今日总时长(分钟)
8. **今日记录**:列表显示每次专注的任务名、时间、时长,可编辑、删除
9. **手动添加**:输入分钟数和任务描述,手动添加一条记录

### 功能清单

| 功能 | 操作 | 说明 |
|------|------|------|
| 选择关联任务 | 点击「关联任务」卡片 | 弹出任务选择器 |
| 开始/暂停计时 | 点击「开始专注」/「暂停」 | 25分钟倒计时 |
| 重置计时 | 点击「重置」(有进度时显示) | 恢复到设定的专注时长 |
| 调整专注时长 | 点击 +/- 按钮或直接输入 | 5-120分钟范围 |
| 调整休息时长 | 点击 +/- 按钮或直接输入 | 1-30分钟范围 |
| 自动休息 | 专注结束后自动开始 | 休息时间到后弹出提示 |
| 完成任务关联更新 | 自动 | 专注完成后自动累加任务的 actual_duration |
| 农场联动 | 自动 | 完成后自动给对应科目浇水 |
| 手动添加记录 | 输入分钟数+描述→点「添加」 | 不在计时器中的学习时间也可以记录 |
| 编辑记录 | 点击记录右侧「编辑」 | 修改时长和任务名 |
| 删除记录 | 点击「删除」 | 二次确认后删除 |

### 完成通知

番茄钟完成一轮后:
- **Web Audio API**:播放上行音阶(C5→E5→G5)
- **振动**:H5 用 `navigator.vibrate()`,App 用 `uni.vibrateLong()`
- **浏览器通知**:页面在后台时弹出系统通知
- **后端同步**:自动创建 FocusRecord 到数据库

### 统计接口

后端提供 4 个统计维度,所有数据来自 `focus_records` 表:
- `/api/focus/stats` — 总览(总时长、番茄数、平均时长)
- `/api/focus/stats/subject` — 按科目聚合
- `/api/focus/stats/daily` — 按日聚合
- `/api/focus/stats/weekly` / `/monthly` — 按周/月聚合

## 6. 复习模式(卡片和错题共用)

### 卡片复习

1. 点击「开始复习」进入
2. 显示进度:**3 / 10**(第3张,共10张)
3. 每次显示一张卡片:
   - 科目标签
   - **Q**:问题文字 + 问题图片(可点击放大)
   - 「点击查看答案」按钮
4. 点击后显示:
   - **A**:答案文字 + 答案图片
5. 底部出现三个反馈按钮:
   - 😣 **未掌握** → 明天再复习
   - 🤔 **较熟悉** → 3天后复习
   - 😎 **已掌握** → 7天后复习
6. 复习完所有卡片显示 🏆 "复习完成!",可点击返回列表

### 错题复习

1. 类似卡片,显示:
   - 科目标签
   - **Q**:题目 + 图片
   - 「点击查看答案」
2. 查看答案后额外显示:
   - **分析**:错误分析文字
3. 两个反馈按钮:
   - ❌ **做错了** → 正确次数清零,明天再复习
   - ✅ **做对了** → 正确次数+1,连续3次正确标记为已掌握
4. 复习完成显示 🏆 "复习完成!" + 正确/总题数

## 7. 导出功能

点击页面右上角「📤 导出」按钮,弹出导出弹窗:

### 导出选项

| 格式 | 生成方式 | 特点 |
|------|---------|------|
| **CSV** | 下载 .csv 文件 | 可用 Excel / Numbers 打开;UTF-8 BOM 编码兼容中文 |
| **Excel** | 下载 .xls 文件(HTML 表格) | 保留颜色样式;图片以 `<img>` 标签嵌入 |
| **PDF** | 调用浏览器打印对话框 | 另存为 PDF;横向 A4 排版 |

### 导出筛选

- 导出的数据遵守当前页面所有筛选条件(科目、标签、掌握程度、做错次数)
- **包含答案开关**:关闭后只导出问题,不导出答案列
- **智能列**:如果所有记录都没有文字内容(纯图片卡片),则不显示"问题"列,只显示"问题图片"列

### 图片导出

- 图片以 base64 data URL 格式嵌入
- Excel/PDF 中显示为 120×80px 缩略图
- CSV 中显示为图片 URL(管道符 `|` 分隔多个图片)

## 8. 学习统计

### 布局概览

1. **时间范围切换**:今日 / 本周 / 本月 / 全部
2. **概览卡片**(2×2):总专注分钟、完成番茄数、记录条数、平均每次时长
3. **🍩 任务时间分布饼图**:SVG 环形图,每种任务不同颜色,中央显示总分钟数
4. **📚 科目分布**:各科目进度条(颜色+百分比+分钟数+专注次数)
5. **📈 学习趋势图**:柱状图,Y轴自适应刻度,每日/每周/每月粒度
6. **📋 最近记录**:最近 15 条聚合记录(日期+任务名+次数+分钟数)

### 数据来源

所有统计数据均来自后端数据库的 `focus_records` 表,通过 `/api/focus/stats*` 系列接口获取,不是 localStorage。换设备登录同一账号可看到相同的统计数据。

## 9. 学习农场

### 游戏机制

| 操作 | 来源 | 效果 |
|------|------|------|
| 种下种子 | 首次完成某科目的番茄钟或任务 | 自动创建该科目的植物 |
| 浇水 | 完成该科目的番茄钟 | +15% 成长值,+5 XP |
| 施肥 | 完成该科目的任务 | +30% 成长值,+12 XP |
| 收获 | 植物达到 100% 成长度 | +50 金币,+20 XP |

### 成长阶段

| 阶段 | 进度 | 图标 | 外观 |
|------|------|------|------|
| 种子 | 0-29% | 🌱 | 灰色边框 |

## 11. AI 智能规划

### AI Provider 优先级

后端通过环境变量自动选择(代码见 [services/ai_service.py](studymate-uniapp/server/services/ai_service.py)):

```
HUNYUAN_API_KEY 已配置 → 用腾讯混元
否则 DEEPSEEK_API_KEY 已配置 → 用 DeepSeek
否则 → 用 mock(内置假数据,演示用)
```

可强制指定:`AI_PROVIDER=hunyuan|deepseek|mock`

### 两个模式

**1. 整体规划对话**
- 输入你的考试信息(如"考研408,150天,每天8小时")
- AI 生成包含阶段划分、每周目标、每日时间分配、复习策略的完整计划
- 可以多轮对话调整
- 满意后点击「确认并应用计划」直接创建

**2. 科目框架分析**
- 选择科目 → 上传教材目录图片 → 通义千问 Vision AI 识别章节
- AI 给出每章建议学习时间(分钟/天)和预计天数
- 可进一步对话调整章节安排
- 确认后一键生成今日任务

### AI 接口列表

| 接口 | 功能 |
|------|------|
| `POST /api/plans/ai/generate` | 生成完整学习计划 |
| `POST /api/tasks/ai/generate` | 生成今日任务 |
| `POST /api/cards/ai/generate` | 从学习内容生成卡片 |
| `POST /api/ai/review` | 生成每日复盘总结 |
| `POST /api/ai/syllabus` | 识别教材目录图片(通义千问 Vision) |

> 未配置任何 AI API Key 时,文本生成类接口使用内置 Mock 数据演示功能;图片识别接口返回 mock 的章节列表。

## 13. 图片上传

### 三种上传方式

| 方式 | 适用平台 | 说明 |
|------|---------|------|
| 📷 **拍照** | 手机端 | 调用系统相机,单张拍摄 |
| 🖼️ **相册** | 所有平台 | 从相册选择,最多 9 张 |
| 📋 **粘贴** | H5 电脑端 | 点击「粘贴」按钮后按 Ctrl+V,从剪贴板读取图片 |

### 图片处理与存储

两种存储模式(由后端是否配置 COS 决定):

**模式 A:base64 数据库存储(默认)**
- 图片自动转为 base64 data URL(JPEG,质量 75%,最大 1200px)
- 存入数据库的 JSON 数组字段(`question_images` / `answer_images`)
- 优点:零配置,跨端同步天然支持
- 缺点:图片多则数据库体积膨胀

**模式 B:腾讯云 COS 直传(生产推荐)**
- 后端配置 `COS_SECRET_ID` / `COS_SECRET_KEY` / `COS_BUCKET` / `COS_REGION` 后启用
- 前端先调 `/api/upload/sts` 拿临时凭证,再直传到 COS
- 优点:不占数据库,CDN 加速
- 缺点:需要配置 COS 存储桶 + CORS 跨域规则

### 显示

- 列表页显示缩略图(60×60px),点击可全屏预览
- 导出时 base64 图片直接嵌入 Excel/PDF;COS 图片以 URL 形式显示

## 14. 艾宾浩斯记忆算法

### 知识卡片

| 掌握程度 | 第1次 | 第2次 | 第3次 | 第4次 | 第5次 | 第6次+ | 长期 |
|---------|------|------|------|------|------|--------|------|
| 未掌握 | 1天 | 1天 | 2天 | 3天 | 5天 | 8天 | 30天 |
| 较熟悉 | 3天 | 5天 | 8天 | 14天 | 21天 | 30天 | - |
| 已掌握 | 7天 | 14天 | 30天 | 30天 | 30天 | 30天 | - |

**规则**:
- 复习时回答正确 → 掌握程度升一级(未掌握→较熟悉→已掌握)
- 回答错误 → 降一级(已掌握→较熟悉→未掌握)
- 下次复习日期 = 上次复习日期 + 对应天数
- 系统保证**下次复习日期永不低于创建日期**

### 错题本

| 连续正确次数 | 下次复习间隔 |
|------------|------------|
| 0次(刚做错) | 明天 |
| 第1次正确 | 1天后 |
| 第2次正确 | 3天后 |
| 第3次正确 | 7天后 |
| 第4次正确 | 14天后 |
| 第5次正确 | 30天后 |

**规则**:
- 做错时正确次数清零,错误次数+1
- 连续正确 ≥3 次 → 自动标记为"已掌握"

## 15. 多端同步与部署架构

### 数据同步

- **数据库**:所有任务、卡片、错题、番茄记录存储在 PostgreSQL 数据库
- **图片**:base64 存数据库 JSON 字段 或 腾讯云 COS 对象存储(后端配置 COS 后启用)
- **用户设置**:科目列表存数据库;番茄钟时间设置和暗色模式存浏览器 localStorage
- **登录态**:JWT Token 存 localStorage

### 当前部署架构(Vercel 双端)

```
用户手机/电脑浏览器
        ↓
   Vercel 前端(静态 H5)
        ↓ /api
   Vercel 后端(Python Serverless,FastAPI + Mangum)
        ↓
   ┌────┴────────┐
   ↓             ↓
 Supabase PG   腾讯云 COS
```

- **前端**:Vercel 从 GitHub 自动构建,产物 `dist/build/h5`
- **后端**:Vercel Python Serverless Functions,入口 `server/api/index.py`
- **数据库**:Supabase 免费 PostgreSQL(500MB)
- **图片**:腾讯云 COS(可选)
- **限制**:Vercel Serverless 10 秒超时,AI 长接口可能超时

### 后续规划:CloudBase + 小程序

```
电脑浏览器(H5,Vercel)─→ CloudBase HTTP 云函数(FastAPI)
                                  ↓
微信小程序(手机)    ─→  调混元 AI(1亿 Token 福利)
                                  ↓
                            Supabase PG + COS
```

利用微信「AI 小程序成长计划」福利:
- 6 个月免费 CloudBase 环境(部署后端,无 10 秒超时限制)
- 1 亿混元 Token(AI 接口,通过云函数内调用消耗福利)
- 1 万张混元文生图(备用)

后端代码已适配 CloudBase,入口为 [server/index.py](studymate-uniapp/server/index.py),生产环境检测会自动开启(`TENCENTCLOUD_RUNENV` 等环境变量)。

详细教程见 [docs/cloudbase-deploy.md](studymate-uniapp/docs/cloudbase-deploy.md)。

### 后续规划:云服务器

迁移到腾讯云轻量服务器(60-100 元/年),Docker Compose 一键拉起:
- Nginx(前端静态 + 反代 `/api`)
- FastAPI(uvicorn + systemd 或 Docker)
- PostgreSQL(本地或继续用 Supabase)

优点:无超时限制、国内访问快、年费用最低。

### 访问方式

| 端 | 当前 | 规划 |
|----|------|------|
| 电脑 | 浏览器打开 Vercel 域名 → 收藏夹 | 同左,或自定义域名 |
| 手机 | 浏览器打开 Vercel 域名 → 添加到主屏幕 | 微信小程序 |
| 数据同步 | 同一账号登录,所有数据自动同步 | 同左 |
